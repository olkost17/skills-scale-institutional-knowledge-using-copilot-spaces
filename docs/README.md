# OctoAcme Project Management Processes

Welcome to OctoAcme’s project management documentation. This folder centralizes our approach and practical guidance for planning, delivering, and improving our projects.

## Overview

OctoAcme applies a clear and flexible framework for project management designed to deliver customer value and consistent results. We follow a five-phase lifecycle: **Initiation**, **Planning**, **Execution**, **Release**, and **Close & Retrospective**. Each phase includes specific deliverables—from the initial Project One-pager through to retrospectives and lessons learned—ensuring all projects are launched with clear goals, tracked transparently, and improved iteratively.

## Key Workflows

- **Team Rhythm**: Daily standups to surface progress and blockers, weekly delivery syncs to discuss updates and risks, and regular demos at sprint or milestone completion.
- **Work Tracking**: Use of GitHub Projects boards with standardized columns (Backlog, Ready, In Progress, In Review, QA, Done) and a disciplined pull request process (small PRs, linked issues, acceptance criteria, code review).
- **Quality Assurance**: Automated testing (unit, integration, end-to-end), linting, security scanning, and manual QA for key features before release. Release readiness is ensured by CI passes, drafted release notes, and rollback plans.
- **Risk & Communication**: Risks are recorded in a register and reviewed weekly; blockers are escalated in three levels—team standup, PM escalation, sponsor escalation. Stakeholder communication follows a consistent cadence—weekly PM/Product sync, biweekly standups, monthly status updates, and as-needed escalations.

## Personas & Roles

- **Product Managers (PdM)**: Define outcomes, prioritize the backlog, and own success measurement.
- **Project Managers (PM)**: Coordinate schedules, risks, and communication, and steer progress across teams.
- **Developers**: Implement features, ensure testability, and uphold code quality.
- **QA/Testing**: Validate that deliverables meet acceptance criteria, run tests, and enable smooth releases.

## Continuous Improvement

After each release or major milestone, teams run retrospectives focusing on what went well, what can improve, and identifying 2–3 prioritized action items for process enhancement. These are tracked for follow-up, promoting a blameless, learning-focused culture.
